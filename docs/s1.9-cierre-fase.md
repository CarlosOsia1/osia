# S1.9 — Cierre de Fase 1: hecho local vs. pendiente externo

> Estado al cierre de la sesión. Los **gates locales** quedaron implementados y verdes en CI; los
> ítems **externos** necesitan tus cuentas/decisiones y NO los puede hacer Claude.

## ✅ Hecho local (sin cuentas, ya en el repo)

- **CI (`.github/workflows/ci.yml`)** — install · lint · typecheck · test · build con cache de Turbo,
  en push/PR a `main`. (Ya existía; ahora sus gates incluyen los dos tests nuevos de abajo.)
- **Gate de contraste WCAG AA** (`packages/ui/src/contrast.test.ts`) — parsea los tokens reales de
  `styles.css` y falla el CI si un par texto/fondo baja de AA. **Verde** (champán/marfil/taupe sobre
  ónix cumplen).
- **Contract test de `ErrorCode`** (`packages/shared/src/contracts.test.ts`) — red de seguridad en
  runtime contra drift del enum (key === value); el sobre de error de apps/api ya es type-safe.
- **Security headers** — `apps/api` (helmet: nosniff/frame DENY/HSTS + Permissions-Policy + CORS
  allowlist + `x-request-id`), `apps/web` y `world-server` ya los tenían; se **completaron en
  `world-client`** (nosniff, frame DENY, referrer, Permissions-Policy `mic=self`).
- **`requestId`** correlacionado en el `ApiError` (Pino) — ya presente.
- **Presence DB-opcional** — el world-server corre sin `DATABASE_URL`; con él, checkpointea presencia.

## ⛔ Pendiente externo (necesita tus cuentas / decisiones)

### S1.9-H1 — Hardening de borde
- [ ] **Cloudflare**: dominio en CF, WAF managed ruleset, Bot Fight Mode, rate-limit en `/api/auth/*`,
      proxy naranja (oculta IP de Hetzner). R2/CDN para assets futuros.
- [ ] **Redis** (Upstash/Hetzner): `REDIS_URL` → `RateLimitGuard` de negocio (`rl:auth`, `rl:invite`,
      `rl:upload`) vía Lua atómico. (Hoy: rate-limit in-process en world-server; falta el de apps/api.)
- [ ] **Doppler**: secrets centralizados → inyección a Vercel + Hetzner (nada de secrets en bundles).

### S1.9-H2 — Observabilidad
- [ ] **Sentry** (free): proyectos `web` / `api` / `world-client` → `SENTRY_DSN` en env + source maps
      en el build (Vercel/Docker). Correlación `requestId` Pino ↔ Sentry.
- [ ] **Discord**: webhook a `#alerts` (error rate, DB > 400MB, world-server caído).
- [ ] Health-check de Supabase (evitar pausa por inactividad).

### S1.9-H3 — CI/CD de deploy
- [ ] **GitHub repo secrets** (`SUPABASE_*`, deploy tokens) para el job `supabase db diff` (drift de
      migraciones) y los deploys.
- [ ] **Vercel**: proyectos para `web` + `world-client` (deploy por subdominio).
- [ ] **Hetzner**: Docker para `api` + `world-server`.

### S1.9-H4 — SSO local idéntico a prod
- [ ] **`*.osia.localhost`**: hosts + dev proxy (`osia.localhost`, `mundo.osia.localhost`,
      `auth.osia.localhost`). Cookie `Domain=.osia.localhost` por entorno. Logout global vía
      revocación + pub/sub Redis verificado entre subdominios.
